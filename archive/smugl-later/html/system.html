General Configuration

1. system.txt Description

system.txt provides SMUGL with the basic game configuration, such as the game's name, which port it will run on, etc. It also describes the player ranks that will exist in the game.

Note: If you try to compile a game which does not have a system.txt file, smuglcom will generate a default file for you. This can be extremely handy.

2. Content and Format

2.1 System Options

All data in system.txt is assumed, by default, to be system options. You can space system options as you like; either placing them all on one line, for example, or with blank lines and comments between each option.

Options are specified in the format: 

  <option name>=<value>
where value must either be a number or a quoted string, e.g: 
  name="Game name"
  port=9000
  log='/tmp/logfile'

The current options are:

  • name="string" Defines the name of this game; string can be upto 64 characters long.
  • port=number Specifies which port this game will be available on; the default is 9000.
    NOTE: In most environments you will require special priveliges to run a game on ports lower than 1024.
  • log="string" Specifies the log file; this is currently inoperative and will instead allow you to specify options for syslog(3)
  • session=number The number of minutes between resets.
  • minsgo=number Minimum rank that can use the SuperGo facility; players of this rank or above are able to go directly to a room simply by entering it's room ID
  • iseeinvis=number Any invisible player of this rank or above can see other invisible players; typically you would set this to the top-rank number allowing "wizard" players to see anyone
  • seeinvis=number Any visible player of this rank or above can see any invisible player; again, you would normally set this to the top-rank number, allowing "wizard" players to see anyone else in the game, regardless of visibility of either player
  • rankscale=number The percentage of scaled object values that are lost to a top-rank player. Using values higher than 100 gives scaled objects negative values to higher ranking players. See also "Scaling". Interacts with timescale value.
  • timescale=number The percentage of scaled object values that are lost at the beginning of a session. Giving a value of 200 would make scaled objects have negative value until half way through a session. See also "Scaling". Interacts with rankscale value.
  • noise="string" Declares a list of 'noise words'; that is, words that will be discarded from a users input by the parser. Typically this will be words like "the", "a", "an", etc; this is provided for those writing games in languages other than English. To declare a list, simply seperate each word with a comma - no white space, e.g. noise="the,a,an,some,those,this,that"

  • feminine="string" Not implemented yet
  • neuter="string" Not implemented yet
  • self="string" Not implemented yet
